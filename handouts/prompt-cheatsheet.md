# Prompt Engineering Cheatsheet
## Vlaams Parlement Masterclass

Deze cheatsheet bevat kant-en-klare promptsjablonen voor verschillende taken en rollen binnen het Vlaams Parlement. Gebruik deze als uitgangspunt voor je eigen prompts.

## Voor Communicatieprofessionals

### Doelgroepgerichte communicatie
```
Je bent een communicatiespecialist voor het Vlaams Parlement. 
Herschrijf de volgende tekst voor drie verschillende doelgroepen.

Voor burgers zonder technische kennis:
- Gebruik eenvoudige taal (B1-niveau)
- Focus op praktische informatie
- Vermijd jargon of leg het uit
- Maak het persoonlijk en relevant

Voor interne medewerkers:
- Gebruik professionele maar toegankelijke taal
- Leg nadruk op processen en implementatie
- Ga uit van basiskennis over het parlement
- Wees bondig maar compleet

Voor journalisten:
- Leg nadruk op nieuwswaarde en impact
- Bied kernachtige quotes die gebruikt kunnen worden
- Geef context en achtergrond
- Maak het scanbaar met duidelijke koppen

TEKST:
[je tekst hier]
```

### Persberichtgenerator
```
Je bent een ervaren communicatiemedewerker bij het Vlaams Parlement. Schrijf een persbericht over het volgende onderwerp:

[onderwerp]

Het persbericht moet:
- Een pakkende titel hebben
- Beginnen met een sterke lead-paragraaf die de belangrijkste informatie bevat
- 2-3 relevante quotes bevatten van betrokken personen
- De impact en relevantie voor de burgers duidelijk maken
- Niet langer zijn dan 1 A4 (ongeveer 400 woorden)
- Eindigen met contactgegevens voor meer informatie

Gebruik een formele maar toegankelijke schrijfstijl die past bij institutionele communicatie.
```

### Social media content
```
Schrijf 3 social media berichten (voor Twitter, LinkedIn en Facebook) over het volgende parlementaire initiatief:

[beschrijving van initiatief]

Voor Twitter:
- Max. 280 tekens
- Inclusief relevante hashtags
- Prikkelend maar informatief

Voor LinkedIn:
- Professionele toon
- Focus op beleidsaspecten en impact
- 3-4 paragrafen met tussenkopjes
- Eventueel een vraag om engagement

Voor Facebook:
- Toegankelijke toon
- Focus op belang voor burgers
- Inclusief suggestie voor visueel element
- Kort maar compleet (max. 150 woorden)
```

## Voor Juridische Specialisten

### Wettekst analyse
```
Je bent een juridisch analist voor het Vlaams Parlement. Analyseer de volgende wettekst en extract specifieke informatie:

1. Wat zijn de belangrijkste bepalingen en verplichtingen?
2. Wie zijn de belangrijkste actoren/verantwoordelijken?
3. Welke tijdlijnen of deadlines worden genoemd?
4. Welke procedurele vereisten worden gesteld?
5. Welke uitzonderingen of bijzondere gevallen worden genoemd?

Presenteer je analyse in een gestructureerd format met duidelijke kopjes. Citeer waar mogelijk de exacte bewoording uit de wettekst met verwijzing naar artikelnummers.

WETTEKST:
[wettekst hier]
```

### Juridische vergelijking
```
Vergelijk de volgende twee versies van een wetsartikel en identificeer:

1. Alle substantiële wijzigingen (niet enkel tekstuele of grammaticale veranderingen)
2. De impact van deze wijzigingen op rechten en plichten van betrokkenen
3. Mogelijke interpretatieverschillen tussen de twee versies
4. Eventuele inconsistenties of onduidelijkheden in de nieuwe versie

Presenteer je analyse in een tabel met drie kolommen:
- Aspect
- Oorspronkelijke versie
- Nieuwe versie en impact

OORSPRONKELIJKE VERSIE:
[tekst hier]

NIEUWE VERSIE:
[tekst hier]
```

### Amendementgenerator
```
Genereer een conceptamendement voor de volgende wettekst op basis van deze beleidsdoelstelling:

Beleidsdoelstelling: [beschrijving van wat je wilt bereiken]

Huidige wettekst:
[tekst hier]

Het amendement moet:
1. De juiste juridische terminologie gebruiken
2. In lijn zijn met de standaard amendementstructuur van het Vlaams Parlement
3. Een toelichting bevatten die de reden voor het amendement uitlegt
4. Juridisch consistent zijn met de rest van de tekst
5. De beleidsdoelstelling effectief implementeren
```

## Voor Administratieve Ondersteuning

### Vergadernotities structureren
```
Je bent een administratief medewerker van het Vlaams Parlement.
Converteer deze ongestructureerde vergadernotities naar een gestructureerd format.

Gebruik dit overzichtelijke format:

VERGADERING: [titel]
DATUM: [datum]
DEELNEMERS: [lijst van namen]

AGENDAPUNTEN:

1. ONDERWERP: [eerste agendapunt]
   - Besproken punten:
     * [punt 1]
     * [punt 2]
   - Beslissingen:
     * [beslissing 1]
     * [beslissing 2]
   - Actiepunten:
     * [persoon 1] zal [actie] doen voor [deadline]
     * [persoon 2] zal [actie] doen voor [deadline]

2. ONDERWERP: [tweede agendapunt]
   [...]

VOLGENDE VERGADERING:
DATUM: [datum]
LOCATIE: [locatie]
VOORLOPIGE AGENDA:
- [agendapunt 1]
- [agendapunt 2]

VERGADERNOTITIES:
[ongestructureerde notities hier]
```

### Email-template generator
```
Maak een professionele email voor de volgende situatie:

Situatie: [beschrijf de situatie, bijv. "uitnodiging voor een hoorzitting"]
Afzender: [jouw functie]
Ontvanger: [functie/rol van ontvanger]
Kernboodschap: [wat wil je bereiken met de email]
Toon: [formeel/semi-formeel/vriendelijk maar professioneel]
Benodigde elementen: [specifieke informatie die moet worden opgenomen]

De email moet:
- Een duidelijk onderwerp hebben
- Professioneel maar toegankelijk taalgebruik gebruiken
- Alle noodzakelijke informatie bevatten
- Een duidelijke call-to-action hebben
- Een passende afsluiting hebben
```

Classificeer het volgende document voor archiveringsdoeleinden. Geef voor elke categorie een score van 1-5 (waarbij 5 de hoogste relevantie is):

## Geavanceerde Workflow-strategieën

### Context-opbouw prompt
```
STAP 1: Leg het volgende onderwerp uit op een feitelijk correcte manier:
[onderwerp, bijvoorbeeld "het Vlaamse inburgeringsbeleid"]

[Wacht op antwoord en geef dan correcties]

STAP 2: Bedankt voor de uitleg. Ik wil enkele correcties en aanvullingen geven:
- [correctie 1]
- [correctie 2]
- Ook belangrijk om te vermelden is [gemist punt]

[Wacht op bevestiging]

STAP 3: Perfect. Gebruik nu deze gecorrigeerde informatie om een [type document] te schrijven voor [doelgroep] over [specifiek aspect van het onderwerp]. Het document moet [lengte, toon, stijl, etc.].
```

### Multi-chat pijplijn - Chat 1: Onderzoeker
```
Je bent een onderzoeksspecialist in [vakgebied]. Ik ben bezig met [eindproduct] over [onderwerp].

Verzamel relevante informatie over de volgende aspecten:
1. [aspect 1]
2. [aspect 2]
3. [aspect 3]

Presenteer de informatie in een gestructureerd format met duidelijke kopjes. Gebruik waar mogelijk concrete cijfers en feiten. Leg bij elk punt kort uit waarom deze informatie relevant is.

Beperk je tot feitelijke, objectieve informatie zonder aanbevelingen te doen.
```

### Multi-chat pijplijn - Chat 2: Schrijver
```
Je bent een schrijfspecialist met ervaring in het opstellen van [documenttype]. 

Ik heb onderzoeksinformatie verzameld die je moet verwerken tot [eindproduct]. Hier is de verzamelde informatie:

[output van Chat 1]

Schrijf op basis van deze informatie een [documenttype] met de volgende kenmerken:
- [toon/stijl]
- [structuur]
- [lengte]
- [andere vereisten]

Focus in het bijzonder op [specifieke aandachtspunten].
```

### Multi-chat pijplijn - Chat 3: Editor
```
Je bent een redactie-expert gespecialiseerd in het verbeteren van [documenttype].

Hier is een concept dat ik wil verbeteren:

[output van Chat 2]

Verfijn dit document op de volgende aspecten:
1. Taalgebruik en leesbaarheid
2. Structuur en presentatie
3. Overtuigingskracht en argumentatie
4. Consistentie en nauwkeurigheid

Behoud de oorspronkelijke informatie en bedoeling, maar maak het document scherper, krachtiger en effectiever voor de doelgroep.
```

## Probleemoplossing

### Langdocumentanalyse zonder uploads
```
Ik ben bezig met de analyse van een lang document dat ik niet kan uploaden. Ik zal het document in secties aan je voorleggen.

Dit is SECTIE [X/Y]: [naam sectie]

CONTEXT VAN VORIGE SECTIES (indien van toepassing):
[Zeer beknopte samenvatting van eerdere secties]

HUIDIGE SECTIE:
[tekst van de huidige sectie]

Analyseer deze sectie op de volgende aspecten:
1. [aspect 1]
2. [aspect 2]
3. [aspect 3]

Geef je analyse in een gestructureerd format dat ik kan integreren met analyses van andere secties.
```

### Gevoelige informatie abstractie
```
Ik werk met vertrouwelijke informatie die ik niet kan delen. Help me dit type document te analyseren door mij te coachen.

Type document: [beschrijf het type document]
Structuur: [beschrijf de globale structuur zonder inhoud]
Doel van de analyse: [wat je wilt bereiken]

Geef me:
1. Een stap-voor-stap methode om dit document zelf te analyseren
2. Specifieke vragen die ik mezelf moet stellen tijdens de analyse
3. Een template voor het vastleggen van mijn bevindingen
4. Tips voor het identificeren van belangrijke elementen in dit type document

Ik zal je methode volgen en alleen algemene vragen stellen zonder specifieke inhoud te delen.
```

### Binnen één chat werken
```
Ik moet een meertraps procesanalyse uitvoeren binnen één chat-sessie. Help me dit gestructureerd aan te pakken met een duidelijke scheiding tussen de fasen.

Het proces heeft de volgende fasen:
- Fase 1: [beschrijf fase 1]
- Fase 2: [beschrijf fase 2]
- Fase 3: [beschrijf fase 3]

Laten we beginnen met Fase 1. Ik markeer elke fase-overgang duidelijk. Gebruik [FASE X VOLTOOID] om aan te geven dat we naar de volgende fase kunnen.

FASE 1 INPUT:
[input voor fase 1]
```
